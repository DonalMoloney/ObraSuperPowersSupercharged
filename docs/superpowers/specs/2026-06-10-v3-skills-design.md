# v3 First Population — Design

**Date:** 2026-06-10
**Status:** Approved by user (design review), pending spec review
**Scope:** 12 new v3 skills — the first content in the tier

## Goal

Populate the empty v3 tier ("crazy / experimental AI ideas") with a themed first
round of skills, per the v3 rules: creativity over polish, `tier: v3` +
`status: experimental` frontmatter, and a "Why this might be crazy enough to work"
section in every skill. The tier's bar is "captured well enough to evaluate
later" — these are idea captures, not working machinery.

## How the round was shaped

The user chose theme-driven exploration across four themes, and after reviewing
3 ideated candidates per theme (generated by the `moonshot-ideator` agent per the
project workflow), selected ALL 12 rather than one per theme. v3's low quality
bar makes a wide capture legitimate; filtering happens later at graduation time.

## The twelve skills (`v3/skills/<name>/SKILL.md`, flat layout)

### Theme: Self-improving skills

1. **skill-darwin** — Maintain 2–4 variant phrasings of a high-traffic skill;
   randomly assign one per session; score fitness (first-try verification, count
   of user corrections) in a ledger; mutate the losing variants. *Risk to record:
   fitness signal is hopelessly noisy at small n — may evolve coin-flip winners.*
2. **skill-scar-tissue** — After a failure or user correction, graft a dated,
   quarantined `## Scar` rule onto the skill that should have prevented it.
   Recurring scars promote into the skill body; never-firing scars decay and are
   pruned. *Risk: blame misattribution turns every skill into a junk drawer of
   irrelevant warnings.*
3. **skill-cannibal** — Periodic "metabolism" pass: skills compete for a fixed
   token budget; underperformers get eaten (useful lines absorbed into a stronger
   skill, husk archived); habitually co-loaded skills may fuse. *Risk: frequency
   is a terrible proxy for value — may eat the rarely-used fire extinguisher.*

### Theme: Agent swarms & ecology

4. **agent-bazaar** — For plans with 4+ parallelizable tasks, an auctioneer posts
   each task and 3–5 bidder subagents submit sealed approach+cost bids; cheapest
   credible bid wins; blown estimates incur reputation penalties in a ledger.
   *Risk: same-model bidders make the economy theater — selects for confident
   hallucinated estimates.*
5. **predator-prey-review** — Prey agents write code plus a private
   hidden-weaknesses note; predator agents score only for bugs NOT on the note;
   both lineages' playbooks ratchet from the kill log each generation. *Risk:
   Goodharting — exotic non-issues and decoy bugs optimize for drama over
   correctness.*
6. **parliament-of-ghosts** — Five persistent persona-agents (e.g.
   Maintainer-in-Five-Years, Security Paranoiac, Intern Who Inherits This, User
   at 3 AM, Accountant) debate flagged-irreversible decisions and vote; each
   persona's cross-session track record grows or shrinks its voting weight;
   outvoted personas file dissents re-read on later failure. *Risk: one model
   wearing five hats may be a one-voter parliament.*

### Theme: Memory & long-horizon learning

7. **project-hippocampus** — Session-end consolidation writes 3–5 episodes
   (situation → action → outcome, with file paths and valence); session-start
   recall injects a "last time we touched this area" briefing. Unrecalled
   episodes decay via re-summarization; recalled ones reinforce (Ebbinghaus
   mechanic — self-cleaning by design). *Risk: keyword-grade retrieval briefs
   confidently about the wrong "last time".*
8. **belief-ledger** — Track load-bearing assumptions as probabilistic statements
   ("flaky test is timing-related: 70%"); sessions end by asking what moved;
   beliefs crossing below 50% trigger an audit of decisions built on them.
   *Risk: probabilities are vibes wearing a number costume; audit cascades could
   re-litigate everything.*
9. **inherited-instincts** — A cross-project genome of context-free
   pattern→emotion reflexes ("dread: config that works locally") stored globally;
   instincts strengthen when vindicated and go recessive when wrong in a new
   context; surfaced as gut feelings, never directives. *Risk: deliberately
   engineered prejudice — one wrong dominant instinct biases months of work.*

### Theme: Simulation & counterfactuals

10. **ghost-run** — Before executing a plan, a simulator agent hallucinates the
    entire run (predicted command outputs, file conflicts, test failures) into a
    ghost transcript; real execution diffs against it step by step and halts on
    divergence — a surprised plan is a wrong plan. *Risk: ghost and plan share
    one world-model, so they may share every blind spot.*
11. **premortem-multiverse** — For changes touching data, auth, money, or
    anything irreversible, 3–4 obituary agents each write a six-months-later
    incident report in a distinct doom genre, citing real files; failure chains
    cited by multiple reports become mandatory tests or plan amendments. *Risk:
    generic doom fiction plus ceremony on every scary change.*
12. **branch-historian** — When a project feels stuck, walk the decision history
    backwards, pick 2–3 contested decisions, and actually spike the road not
    taken in throwaway worktrees on a fixed budget; return a regret report that
    vindicates the decision or recommends deliberately re-deciding it. *Risk:
    a 30-minute spike always looks cleaner than accumulated reality — a
    rewrite-everything machine.*

## Per-skill format

- Frontmatter: `name`, `description` (stating WHEN the skill triggers),
  `tier: v3`, `status: experimental`.
- Body sections, in order:
  1. Concept and mechanism — a few paragraphs; loose is fine at this tier.
  2. `## Why this might be crazy enough to work` (mandated by v3/README.md).
  3. `## Known risks / absurdities` — the failure mode named above, recorded so
     future graduation evaluation starts honest.

## v2 boundary lines

Three skills sit near existing v2 identities; each gets an explicit one-line
boundary in its body:

- **belief-ledger** ≠ v2 `decision-ledger`: beliefs/evidence vs decisions made.
- **predator-prey-review** ≠ v2 `red-team-spec`: co-evolving post-implementation
  code adversaries vs one-shot pre-implementation spec attack.
- **branch-historian** composes with v2 `spike-in-worktree` (uses it as its spike
  mechanism) rather than replacing it.

## Explicitly rejected: shared substrate

Five skills imply ledger/hook infrastructure; a shared `v3/_substrate/` was
considered and rejected (YAGNI). Each skill *describes* the files it would keep;
no scripts, hooks, or ledgers are actually built this round. Infrastructure gets
built only when an idea graduates toward v2.

## Build workflow

1. Write the 12 SKILL.md files (order free; all independent — none blocks
   another).
2. Add a "Current skills" table to `v3/README.md` (name + one-line hook),
   mirroring the v2 README pattern.
3. Run the `skill-auditor` agent over the batch, checking v3 frontmatter
   requirements and the mandatory "Why this might be crazy enough to work"
   section. v1/v2 quality bars do not apply.
4. No commit — the project is intentionally not a git repository yet (same
   deviation from the brainstorming default as the v2 spec, per CLAUDE.md).

## Out of scope

- Any executable infrastructure (scripts, hooks, ledgers) for these skills.
- Graduating anything to v2, touching v1/v4/v5, or modifying existing v2 skills.
- A 13th idea or new themes — the round is closed at these 12.
