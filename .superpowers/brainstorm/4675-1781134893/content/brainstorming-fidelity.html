Supercharging brainstorming: how should mockup fidelity work?

Same example screen (a login page) shown the way each option would render it. This decides the headline improvement option for the brainstorming section of the v1 doc.

A

Fidelity Ladder — user asks to level up

Every mockup starts as a fast wireframe. The page itself offers "Show me this in high fidelity" as a clickable action. Spec depth follows: wireframe-only screens get a one-paragraph spec section; screens you leveled up get full visual-detail spec sections (colors, spacing, states).

Pros

  • Cheap by default, expensive only on request
  • Spec depth maps 1:1 to where you invested attention
  • Explicit control — never burns tokens uninvited

Cons

  • Extra round-trip when you do want hi-fi
  • You must know to ask
B

Auto-matched fidelity — skill infers from the question

The skill picks fidelity from the question type: layout/structure questions get wireframes, look-and-feel/polish questions automatically get high fidelity. No asking needed — but also no manual override defined.

"Which layout?" → auto wireframe
"Which feels premium?" → auto hi-fi
Minimal light
Deep dark

Pros

  • Zero friction — right fidelity appears automatically
  • Fewer round-trips than the ladder

Cons

  • Skill can guess wrong and waste tokens on hi-fi you didn't want
  • No explicit lever tying fidelity to spec depth
C

Fidelity contract — set once at session start, drives the whole spec

At companion-offer time the skill asks one extra question: "wireframe pass, or full visual design pass?" That single choice sets mockup fidelity for the session AND the spec template — a visual-design pass produces a spec with a full visual-design chapter (palette, type scale, component states); a wireframe pass produces a lean structural spec.

Session start — one extra question
🩻 Wireframe pass
Fast structural mockups → lean spec (flows + layout only)
🎨 Visual design pass
Hi-fi mockups throughout → in-depth spec with visual-design chapter

Pros

  • Strongest fidelity↔spec-depth coupling — exactly your stated goal
  • Predictable token cost, decided up front

Cons

  • All-or-nothing for the session — can't level up just one screen
  • One more question before brainstorming starts

These can also combine — e.g., C's session default + A's per-screen "level up" escape hatch. Click a card, then tell me in the terminal (including if you want a hybrid).
